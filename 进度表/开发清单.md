# 按模块开发清单

## 模块 A：项目基线与公共能力

目标：把前后端基础框架跑通，形成后面所有模块的底座。

-  整理现有登录框架，确认哪些代码复用，哪些新建
-  后端补齐基础配置：Django、DRF、JWT、MySQL、Redis、媒体文件目录
-  新增统一响应格式：`code / message / data`
-  新增健康检查接口 `/api/health/`
-  配置 Celery 基础入口 `celery.py`
-  前端配置 Axios 请求实例与 token 拦截器
-  搭建前端主布局：侧边菜单、顶部栏、内容区
-  新建仪表盘占位页
-  跑通前后端本地启动流程
-  写一个健康检查测试，确认接口可用

验收：

-  登录后可以进入主布局
-  `/api/health/` 返回 200
-  前后端本地可稳定启动

-  任务失败时记录错误信息
-  文档详情页增加任务状态轮询
-  支持重新入库 / 重建索引
-  写入库链路成功/失败测试

验收：
