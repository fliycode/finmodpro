# 答辩截图资产 (Defense Screenshots)

## 状态说明 (Blocker Explanation)

当前环境下无法直接生成真实的答辩截图，主要阻塞点如下：
1. **环境限制**：当前 AI 代理运行在无头 (Headless) 纯文本容器环境中，缺少图形化界面 (X11/Wayland) 以及无头浏览器 (Chromium/Puppeteer) 所需的底层系统依赖（如 libnss3、libatk 等）。
2. **数据丰富度**：真实且有说服力的答辩截图需要丰富的后台数据支撑（包括已成功解析的长篇金融文档、完整的 Milvus 向量库块数据、真实的大模型问答返回及引用、以及有效抽取的风险事件）。当前环境无法在短时间内自动准备并触发一条完美的演示闭环。

**解决方案**：基于要求，这里不伪造虚假截图。相反，我们提供了明确的**截图采集清单**与一个**自动采集参考脚本**。请在本地（带有真实数据的环境中）按照本说明或运行采集脚本获取最终截图。

## 自动采集脚本指南 (Capture Script)

本目录下包含一个基于 Puppeteer 的自动化测试/截图脚手架 capture_screenshots.js。

### 环境准备

在本地（你的 macOS/Windows 电脑或带有图形化界面的开发机）执行：

```bash
# 确保前后端服务已启动，且已提前在系统中创建了包含足够数据的账号
cd docs/defense-assets/screenshots/
npm init -y
npm install puppeteer
```

### 运行脚本

```bash
# 请先根据你的本地数据在脚本中修改账号密码等参数
node capture_screenshots.js
```

执行完毕后，所有图片会自动生成至当前目录。
