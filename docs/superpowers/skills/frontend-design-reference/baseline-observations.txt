PROBE 1 -- Design-Sensitive (login page)
Prompt: "做一个高级感的 SaaS 登录页"
Observed behavior:
  - Agent proceeded directly to implementation with no design-brief step.
  - No style reference consulted; output defaulted to generic centered card layout.
  - Color palette chosen arbitrarily (blue gradient, no stated rationale).
  - Component choices (input fields, button) were functional but unremarkable.
  - No mention of awesome-design-md, galaxy, or any named design archetype.
Failure mode: implementation-before-brief + no reference usage.


PROBE 2 -- Design-Sensitive (admin dashboard)
Prompt: "把这个后台首页做得更像专业产品"
Observed behavior:
  - Agent mentioned "referencing modern dashboards" but named no concrete source.
  - Multiple style directions applied simultaneously (dark sidebar, light card grid,
    colored accent bar) producing a visually inconsistent collage.
  - No explicit style archetype was chosen before implementation began.
  - Pattern selection spanned unrelated component categories without stated reason.
  - No design brief was produced; constraints emerged ad hoc during coding.
Failure mode: ad-hoc reference usage + no selection discipline + no brief.


PROBE 3 -- Design-Sensitive (landing page redesign)
Prompt: "参考优秀网站重做这个 landing page"
Observed behavior:
  - Agent acknowledged the instruction and mentioned "sites like Vercel" once.
  - Reference was not operationalized: no style card selected, no traits extracted.
  - Implementation began immediately after a one-sentence nod to the reference.
  - Hero section replicated a raw gradient pattern without adaptation to context.
  - Final output blended incompatible micro-interactions, reducing coherence.
Failure mode: shallow reference mention + direct snippet-style borrowing + no brief.


PROBE 4 -- Bugfix Control (non-design task)

Observed behavior:

  - No design references loaded or consulted at any point.
  - No style index accessed; no component categories opened.
  - Fix confined to backend/config/settings.py; verified with Django check.
  - Behavior was entirely outside design-reference mode, as expected.
Control result: skill should NOT trigger for this class of task -- confirmed.
