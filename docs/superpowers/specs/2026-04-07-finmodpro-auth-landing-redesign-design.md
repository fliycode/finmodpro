# FinModPro 登录页重塑设计文档

## 背景

FinModPro 当前登录页已经接入基础品牌信息、登录/注册切换和角色入口，但整体视觉更像功能堆叠页，而不是一个有明确品牌气质、适合展示和答辩演示的产品入口。

本次设计目标不是重写认证流程，而是在保留现有 logo、登录/注册逻辑和工作台入口能力的前提下，重塑登录页的视觉语言、信息层级和动态表现。

## 目标

本次改造需要同时满足两类需求：

- 展示层面：页面具备更强的品牌辨识度，能够在演示、答辩和截图场景中直接代表 FinModPro。
- 使用层面：登录和注册流程仍然清晰高效，不因为装饰性设计而降低输入和切换效率。

最终选择的方向为：

- 视觉气质：品牌门厅感
- 页面重心：展示与效率平衡
- 设计策略：门厅式分层卡片布局，logo 保留并强化为品牌主视觉

## 非目标

- 不修改后端认证接口和表单字段
- 不新增忘记密码、第三方登录、多租户等新业务流程
- 不重构整个应用导航，仅处理未登录态入口页面
- 不替换既有 logo 资产

## 设计方向

### 整体概念

登录页应呈现为一个“品牌门厅”，而不是普通管理后台表单页。

用户进入页面后，第一感受应该是：

- 这是一个可信、克制、精致的金融建模平台
- 品牌资产是清晰的，而不是被表单压缩到边缘
- 登录动作仍然是第一优先级，不需要额外理解成本

### 视觉语气

页面采用浅底、高质感、编辑感较强的视觉路线，避免通用科技产品常见的紫色渐变和默认系统风格。

建议色彩方向：

- 主背景：暖米白、象牙白、浅沙金
- 主文本：深棕黑、墨色
- 强调色：低饱和金棕、香槟金、冷调石墨灰
- 辅助高光：极淡的柔光金或雾蓝，用于层次和氛围，不作为主色

建议字体方向：

- 标题：具有品牌感和展示力的衬线或半衬线字体
- 正文和表单：更稳健、清晰的无衬线字体

目标是形成“品牌展示区更有气质，表单区更易读”的双字体搭配。

## 信息架构

登录页保留现有双态切换能力，但重组布局层级。

### 一级结构

页面由以下三层组成：

1. 背景氛围层
2. 中央主卡层
3. 微交互动效层

### 中央主卡结构

主卡片内部采用左右分区：

- 左侧为品牌叙事区
- 右侧为登录/注册表单区

移动端收敛为单列：

- 先呈现 logo 与品牌说明
- 再呈现表单
- 弱化非关键信息块，避免滚动过长

## 组件设计

### 1. 背景氛围层

背景不使用纯色平铺，应加入轻量、克制的空间感设计：

- 柔和渐变底色
- 模糊光斑或轻微径向高光
- 极淡纹理或噪点
- 少量几何边界线或细分隔纹理

背景的职责是烘托“门厅感”，而不是争夺视觉主角。

### 2. 品牌叙事区

品牌区需要把现有 logo 变成真正的视觉锚点，而不只是插图。

内容建议包含：

- logo
- 产品名 `FinModPro`
- 一句更强的核心价值主张
- 一段简洁说明，解释平台如何连接建模、风险分析、权限治理与智能问答
- 2 到 3 个品牌标签或能力标签

能力标签建议控制在短语级别，例如：

- Financial Modeling
- Risk Intelligence
- Governed Access

现有的功能卡片、检查点列表和权限展示中，只有与未登录态第一屏强相关的内容应该保留。其余大块说明应删除或压缩，避免品牌区再次变成功能说明墙。

### 3. 表单区

表单区继续保留：

- 登录 / 注册切换
- 用户名、邮箱、密码、确认密码
- 服务条款勾选
- 状态提示
- 密码显隐

但视觉上应做以下调整：

- 表单区作为主卡内部的高可读操作区
- 输入框和按钮采用更圆润但更克制的风格
- 登录与注册切换更像内容分段，而不是默认标签页
- 状态消息与错误消息更精确，减少突兀感

交互优先级必须保持明确：

- 标题
- 输入区
- 主按钮
- 辅助文案

### 4. 已登录态承接

当前代码中，未登录显示 `AuthLanding`，已登录显示 `AuthenticatedHome`。本次改造不改变这个分流逻辑。

登录页中与“当前账号状态”“权限标签”“进入工作台”等已登录后信息相关的区块，不应再作为未登录态主要内容出现。

如果该组件仍保留已登录态兼容代码，应确保：

- 未登录第一屏不出现不必要的权限信息
- 已登录后的主承接继续交给 `AuthenticatedHome`

## 动效设计

用户要求增加动态效果，但页面仍然以登录效率为优先之一，因此动效必须“明显但克制”。

### 动效原则

- 首屏有记忆点
- 不影响输入
- 不制造眩晕或注意力干扰
- 优先 CSS 动效，避免不必要的复杂依赖
- 对 `prefers-reduced-motion` 提供降级

### 动效组成

#### 1. 首屏入场

页面加载时执行一次编排式入场：

- 背景光斑轻微浮现
- 主卡整体淡入并向上轻移
- logo 和标题分阶段出现
- 表单区晚于品牌区约 80ms 到 160ms 入场

目标是让页面“被打开”而不是“突然出现”。

#### 2. 背景呼吸感

背景光斑或渐变层做极缓慢的呼吸式漂移，周期较长，运动幅度很小。

目的仅为让页面保持活性，不是制造科技感噪音。

#### 3. logo 微动效

logo 可增加非常轻微的悬浮感、光泽掠过或周边柔光变化。

要求：

- 不旋转
- 不做高频缩放
- 不影响识别

#### 4. 表单交互动效

表单需要更明确的状态反馈：

- 输入框聚焦时边框、高光和阴影变化
- 按钮 hover 时轻微上浮或亮度变化
- tab 切换时使用短时过渡，避免生硬切屏
- 错误提示与状态提示采用柔和淡入

#### 5. 分隔元素动效

品牌标签、分隔线、微型信息块可增加轻量 stagger reveal，但必须只在初次加载触发一次。

### 降级策略

对于低性能设备和减少动态偏好的环境：

- 关闭背景持续漂移
- 关闭 logo 连续动画
- 保留基础 opacity / transform 过渡

## 内容文案方向

页面文案应从“功能堆砌”改成“品牌承诺 + 平台能力摘要”。

建议文案方向：

- 主标题：强调进入工作流或进入智能建模环境
- 副文案：强调连接建模、分析、治理和协作
- 标签文案：短、稳、可截图

避免出现的问题：

- 大段解释性文案
- 过多待办式清单
- 术语密度过高导致首页像产品说明文档

## 技术实现边界

本次改造应尽量限制在前端单页认证入口相关文件内完成。

优先涉及文件：

- `frontend/src/components/AuthLanding.vue`
- `frontend/src/App.vue`
- `frontend/src/style.css`
- 如有必要，可新增局部样式辅助文件或将大段样式内聚到组件内

实现要求：

- 保持现有事件和 props 接口不破坏
- 不改动 `authApi` 调用方式
- 不改动现有登录、注册校验逻辑
- 样式命名继续采用当前组件化类名方式

如果现有 `AuthLanding.vue` 已承担过多视觉和逻辑责任，可以在实现阶段考虑拆出局部纯展示子块，但不做无关重构。

## 响应式要求

### 桌面端

- 双栏布局成立
- 品牌区与表单区存在清晰的视觉重心差异
- 卡片层次、留白和对齐必须足够稳定

### 平板端

- 双栏可保留，但间距和装饰层应收敛
- 品牌区文案长度必须受控

### 手机端

- 切换为单栏
- logo 仍保留，但缩小为更紧凑的主视觉
- 表单按钮和输入框高度适配触控
- 避免首屏信息过高导致用户必须长滚动后才能登录

## 验收标准

本次设计在实现完成后，应满足以下标准：

1. 用户一眼能识别这是 FinModPro 的品牌入口，而不是默认模板页。
2. logo 在第一屏中成为明确视觉锚点。
3. 登录与注册流程不增加额外步骤，输入效率不低于当前版本。
4. 动效存在感明确，但不会妨碍阅读和输入。
5. 页面在桌面与移动端都保持完整结构和视觉质量。
6. 保持现有认证逻辑与页面分流逻辑可用。

## 风险与注意事项

- 过度强调展示感会侵占表单可用空间，需要严格控制品牌区信息量。
- 动效如果过多，会削弱金融平台应有的稳健感。
- 全局样式文件当前存在较多模板遗留变量，实施时需要避免旧样式污染新页面。
- 当前工作区存在未提交改动，实现阶段必须只修改与登录页重塑相关的文件，避免影响其他功能。

## 实施结论

本次登录页改造将采用“品牌门厅感 + 平衡型效率布局”的方案执行：

- 以现有 logo 为主视觉中心
- 以浅色高质感界面重建品牌第一印象
- 以分层卡片结构保障登录效率
- 以克制的首屏与背景动效提升完成度

该方案既适合演示和答辩，也适合继续承接真实登录操作，是当前 FinModPro 登录页最合适的升级方向。
