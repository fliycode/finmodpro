# FinModPro 管理员端优先的双端体验重设计

## 目标
为 `finmodpro` 建立一套统一的产品界面语言，先以管理员端为主视觉基准，再让用户端继承同一设计系统。目标不是单页美化，而是解决当前全站“模板感强、AI 味重、视觉体系不统一、管理员端与用户端割裂”的问题。

本次设计聚焦三个结果：

- 管理员端形成“精致机构终端”风格，具备金融风控后台应有的秩序感、专业度和可信度。
- 用户端沿用同一套视觉与交互规则，但降低信息压迫感，更适合高频日常使用。
- 建立可复用的全局设计骨架，后续新增页面不再靠单页拼装。

## 设计原则
视觉方向选择“精致机构终端”，不用深色大屏，不走营销化渐变风，也不做泛 SaaS 模板。整体基调应是浅色、克制、清晰、报表化、低饱和、强语义。

核心原则如下：

- 统一先于炫技：优先统一布局、字体、状态语义、卡片和导航。
- 管理员端偏决策，用户端偏执行：两端共用视觉语言，但信息密度和操作节奏不同。
- 风险信息必须有优先级：风险、告警、待审、异常不能与普通统计同权重展示。
- 组件服务内容：弱化装饰性 hero、彩色数据卡和伪图表，强化信息层级、对齐、留白和证据链。

## 视觉系统
全局样式从当前偏通用的系统字体与紫色变量，调整为更中性的机构型界面语言。

- 色彩：以冷白、灰蓝、石板灰为主背景与分割色；品牌蓝仅用于主操作和定位；告警橙、风险红、成功绿只用于状态语义。
- 字体：中文优先，标题与正文均保持稳健、现代、无花哨装饰；避免当前偏模板化的通用技术演示气质。
- 背景：以“纸面报表感 + 轻微层次”为主，不再依赖大面积渐变和悬浮玻璃质感。
- 卡片：强调边界、标题、数据对齐、说明文案和状态标签；减少夸张阴影和无意义高饱和色块。
- 图表与指标：如无真实趋势数据，不用装饰性柱条或假趋势；优先展示真实指标、环比、更新时间和异常说明。

## 双端体验分层
管理员端与用户端共用以下基础设计对象：

- 顶层导航结构
- 页面标题区
- 卡片与分区容器
- 表格、筛选、状态标签、按钮
- 间距、圆角、边框、阴影、颜色 token

两端差异如下：

- 管理员端：信息密度更高，强调全局态势、待处理事项、权限与配置、系统运行状态。
- 用户端：信息密度更低，强调任务入口、问答、知识检索、风险摘要和历史工作连续性。
- 管理员端的红橙色状态会更频繁出现；用户端默认保持中性、安静，只有风险内容本身触发强调色。

## 信息架构
### 管理员端
管理员端首页不再是通用 dashboard，而是“决策首页”。建议一级导航按职责分组：

- 总览
- 风险审查
- 知识库
- 模型与评测
- 用户与权限

首页信息优先级定义为：

1. 今日态势与关键动作
2. 风险待处理与异常焦点
3. 平台核心指标与关键趋势
4. 系统状态与运行证据

日志和审计信息应保留，但降级为证据区，而不是首页视觉中心。

### 用户端
用户端首页应从“系统展示”切换为“工作入口”。建议主要入口固定为：

- 智能问答
- 知识检索
- 风险摘要
- 会话历史

用户端首页以三块构成：

- 欢迎与任务入口
- 最近工作与上下文延续
- 系统反馈或关键提示

## 关键页面方案
### 管理员端首页
页面重构为四层：

1. 顶部摘要区：一句态势判断、最近更新时间、两个高优先动作，如“查看待审风险”“刷新运行状态”。
2. 核心指标区：3 到 4 个克制的报表卡，展示关键数据、单位、环比或状态说明。
3. 主工作区：左侧“待处理风险/异常事项”，中间“趋势或负载”，右侧“系统状态摘要”。
4. 证据区：日志、审计线索、最近活动。

禁止继续使用“深色大 hero + 多彩 KPI 卡 + 装饰性小柱状图”的组合。

### 全局导航
侧边栏改为安静、分组清晰、定位明确的机构后台导航。当前项高亮应更克制，通过背景、边框和文字权重体现，不依赖强烈色块。顶栏补充页面说明、当前角色和上下文动作，但不再单独制造大型视觉头图。

### 用户端首页与工作页
用户端首页采用欢迎区、快捷入口、最近工作三段式。问答页强调输入、推荐问题和最近会话；知识库页强调文档、分类与检索；风险摘要页强调结论、证据和来源，不复用管理员式监控卡布局。

## 实施范围
第一阶段先完成共享设计系统与管理员端关键页面，用于验证新方向是否成立。范围包括：

- `sidebar`
- `topbar`
- `page header`
- `section card`
- 状态标签与按钮体系
- 管理员端总览页

第二阶段将规范扩展到用户端首页与核心工作页，确保双端同源但不雷同。

## 非目标
本次重设计不包含以下内容：

- 后端接口改造
- 业务功能重定义
- 无真实数据支撑的复杂图表系统
- 独立品牌重命名或 logo 重做

## 验收标准
设计落地后，应满足以下标准：

- 管理员端首页第一眼更像金融机构后台，而不是通用 SaaS 模板。
- 用户端与管理员端在视觉上同源，但用户端明显更轻、更聚焦任务。
- 全站不再依赖局部 hero 或彩色卡片撑气质。
- 新页面可直接复用统一样式骨架，而不是重复堆定制样式。
- 设计语言能自然覆盖后续页面，而不是只适配当前仪表盘。
